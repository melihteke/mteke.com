I want to give you some idea about how to prepare our Netbox inventory.

Once we install our first Netbox instance, it is a huge workload to add all our real infrasture devices into netbox is the main issue in the begining. And to make it consistent and source of truth, we need to prepare some pipeline. There are some paid synchronozation tools on the market but however I want to give you idea howe can can handle this task without paying premium subscriptions from those tools. In my design I do have 2 main engines. One is Device discovery engine and this engine is reponsible for Discovering network nodes on specific network subnet based on NMAP tool and SNMP protocol.  
  
Step 2 is once we discover a specific ip address is reponding for SNMP then Device discovery Engine collects sysDescryption via MIB value **1.3.6.1.2.1.1.1.0**  
  
Based on retrieved sysDescryption response ( device type) , Device Configuration Check Engine is selected.  
Device COnfiguration Check engin execute some sets of commands to collect information from remote Node.  
  
Afterwards Payload is prepared for Network New device Addition.  
and Finally the new device is added to Netbox.  
  
This is a loop and it runs every X hours/days in our network to make Netbox is consistent with real Infrastructure.
