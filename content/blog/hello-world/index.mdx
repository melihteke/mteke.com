This site is built on **Next.js 16 + MDX + Tailwind v4** and is published to GitHub
Pages. This post is a short tour of the new content format — copy, edit, ship.

## Three theme modes

Use the toggle in the header to switch between **Light**, **Dark**, and **Navy**.
The whole palette — backgrounds, accents, code highlight — re-tunes automatically.

## Colored callouts

<Callout type="tip" title="Tip">
  Spin up a new post with: <code>npm run new-post "Your post title"</code>. It creates
  the folder and a frontmatter skeleton.
</Callout>

<Callout type="info" title="Info">
  Five types are available: <code>info</code>, <code>success</code>, <code>warning</code>,
  <code>danger</code>, <code>tip</code>. Each has its own color and icon.
</Callout>

<Callout type="warning" title="Heads up">
  Relative <code>./image.png</code> paths inside Markdown are automatically rewritten to
  <code>/blog/&lt;slug&gt;/image.png</code>. Just drop images next to the post.
</Callout>

<Callout type="danger" title="Caution">
  Because we ship with <code>output: 'export'</code>, runtime features like
  <code>cookies()</code>, server actions, and request rewrites do not work — everything
  stays fully static.
</Callout>

## Code blocks — language-aware highlighting

Default theme (matches the active site theme — light / dark / navy):

```python title="automation.py" {3,5-7} showLineNumbers
from netmiko import ConnectHandler

def add(a: int, b: int) -> int:
    return a + b

def connect(host: str) -> ConnectHandler:
    """Open an SSH session to a Cisco device."""
    return ConnectHandler(device_type="cisco_ios", host=host)
```

### Per-block alternate themes

Add `style="<name>"` to the fence to opt a single block into a different
syntax-highlight palette. Available: `dracula`, `nord`, `solarized`, `monokai`.

Dracula:

```python title="dracula.py" style="dracula"
def greet(name: str) -> str:
    return f"Hello, {name}!"

print(greet("world"))
```

Nord:

```typescript title="nord.ts" style="nord"
type User = { id: number; name: string };

const users: User[] = [
  { id: 1, name: "Ada" },
  { id: 2, name: "Linus" },
];

users.forEach((u) => console.log(`${u.id}: ${u.name}`));
```

Solarized:

```bash title="solarized.sh" style="solarized"
# Deploy with one command
git add . && git commit -m "post: hello world" && git push
```

Monokai:

```go title="monokai.go" style="monokai"
package main

import "fmt"

func main() {
    fmt.Println("Hello, monokai!")
}
```

Inline syntax: variables like `count{:py}` and strings like `"hello"{:py}` get colored
individually. Plain inline `code` works too.

## Podcast embeds (Apple / RSS / etc.)

  src="https://podcasts.apple.com/us/podcast/the-changelog-software-s-best-weekly-podcast/id341623264"
